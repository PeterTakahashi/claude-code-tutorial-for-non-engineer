# 用語集（比喩つきミニ辞典）

> 勉強会で出てくる言葉を、**やさしい言い換え＋たとえ**でまとめたものです。
> 全部を覚える必要はありません。**「見て驚かない」**ようになれば十分。
> Web の仕組みは、ぜんぶ **レストラン** にたとえて説明します。
>
> 🍽️ レストランのたとえ早見表：
> **客席・メニュー＝フロントエンド** / **ウェイター＝API** / **厨房＝バックエンド** / **食材庫＝データベース** / **店舗＝サーバー** / **店を借りる＝クラウド**

---

## LLM（エルエルエム）

- **ひとことで：** ネットの文章を大量に読んで覚えた“物知りな助手”。
- **たとえ：** たくさんの本を読んだ博識な相談相手。聞けば答えてくれる。
- **もう少し：** Large Language Model（大規模言語モデル）の略。文章を読んで、続きや答えを“それっぽく”作るのが得意。
- **今日の場面：** Claude も Claude Code も、中身はこの LLM。

## コンピュータ

- **ひとことで：** 計算と記憶がものすごく速い機械。
- **たとえ：** 言われた手順を、文句も言わず正確に高速でこなす作業員。
- **もう少し：** スマホもパソコンもサーバーも、中身はぜんぶコンピュータの仲間。
- **今日の場面：** 「サーバー」も結局はあなたの目の前のコンピュータの中で動きます。

## インターネット

- **ひとことで：** 世界中のコンピュータをつなぐ“道”。
- **たとえ：** お店（サーバー）にお客さんが世界中からやってくる道路網。
- **今日の場面：** ngrok でURLを公開すると、この“道”を通って友達がアクセスできます。

## API（エーピーアイ）

- **ひとことで：** プログラム同士が注文をやりとりする“窓口”。
- **たとえ：** レストランの **ウェイター**。お客さん（画面）の注文を厨房（処理）へ運ぶ。
- **もう少し：** 「この形で頼めば、この形で返す」という約束ごと。
- **今日の場面：** 予約フォームの送信ボタンを押すと、API経由で予約が裏側に届きます。

## データベース

- **ひとことで：** データをきちんと整理してしまっておく場所。
- **たとえ：** レストランの **食材庫**。必要なものを出し入れする棚。
- **もう少し：** 表（行と列）でデータを保管・検索できる仕組み。
- **今日の場面：** 受け取った予約（名前・日時・人数）をここに保存します。

## バックエンド

- **ひとことで：** 画面の裏側で、実際の処理をする部分。
- **たとえ：** レストランの **厨房**。お客さんからは見えないけれど、ここで“調理”される。
- **今日の場面：** 「予約を受け取って保存する」のはバックエンドの仕事。

## フロントエンド

- **ひとことで：** 人が見て触る“表側”の画面。
- **たとえ：** レストランの **客席とメニュー**。見た目・操作するところ。
- **今日の場面：** 予約サイトのトップページや予約フォームがこれ。

## クラウドサーバー

- **ひとことで：** 自分で機械を買わず、**借りて使う**サーバー。
- **たとえ：** お店の建物を**買わずに借りる**。使った分だけ払う。
- **今日の場面：** 今日は **使いません**（各自のPCの中で動かすだけ）。常時公開したくなったら、これが必要になる、という名前だけ覚えればOK。

## サーバー

- **ひとことで：** アプリを動かして、リクエストに応える“お店”そのもの。
- **たとえ：** レストランの **店舗**。注文を受けて料理を出す場所。
- **今日の場面：** 今日の「サーバー」は、**あなたのPCの中で一時的に動くだけ**。怖くない。

## Claude（クロード）

- **ひとことで：** Anthropic社が作った AI 助手（LLMのひとり）。
- **たとえ：** 物知りで、頼みごとを理解してくれる相棒。
- **今日の場面：** この Claude に“手”を付けたものが Claude Code。

## Git（ギット）

- **ひとことで：** ファイルの変更履歴を記録する“タイムマシン”。
- **たとえ：** 作業のセーブポイント。いつでも前の状態に戻れる。
- **今日の場面：** 今日はバージョン管理としては深く使いません。ただし **Windowsの人は事前インストールが必要**。理由は git を使うためではなく、**Claude Code（Desktopのローカルセッション）がコマンドを動かす土台（Git Bash）として必要**だから。Macはたいてい標準装備。
- **補足：** Docker や ngrok はGUIで入るので、Gitとは無関係（「Docker/ngrokのためにGit」ではない）。

## GitHub（ギットハブ）

- **ひとことで：** Git で記録したものを、ネット上に置いて共有する場所。
- **たとえ：** 作品やコードを保存・公開する“クラウドの倉庫＆作業場”。
- **今日の場面：** 今日は使いません。世界中のプログラムが集まる場所、という名前だけ。

## CLI（シーエルアイ）

- **ひとことで：** 文字（コマンド）で機械に命令する操作方法。
- **たとえ：** ボタンを押す代わりに、“呪文”を打ち込んで操作する黒い画面。
- **もう少し：** Command Line Interface の略。マウス操作（GUI）の反対。
- **今日の場面：** 今日は **CLIを使いません**。Claude Desktop のアプリ画面（やさしい方）を使います。

## GPU（ジーピーユー）

- **ひとことで：** 同じ計算を一気に大量にこなすのが得意な部品。
- **たとえ：** たくさんの作業員が横並びで同時に手を動かすイメージ。
- **もう少し：** AI（LLM）の学習や動作には、この力がたくさん必要。
- **今日の場面：** 直接は触りません。「AIの裏では大量の計算が走っている」とイメージするための言葉。

## コマンド

- **ひとことで：** コンピュータへの“ひとつの命令”。
- **たとえ：** 「これをやって」という1個の指示書。
- **今日の場面：** 今日は自分でコマンドを打つ場面はほぼナシ。Claude が代わりに実行してくれます。

## MCP（エムシーピー）

- **ひとことで：** AIを外部サービスにつなぐ“コネクタ（差込口）”。
- **たとえ：** AI用の **USB端子**。挿すと、外の道具（ブラウザ・メール・カレンダー等）が使えるようになる。
- **今日の場面：** ハンズオン①で「ブラウザを操作するMCP（Playwright MCP）」を実際に挿して使います。

## Skills（スキル）

- **ひとことで：** よく使う“作業手順書”をまとめたパック。
- **たとえ：** 料理のレシピ集。毎回ゼロから説明しなくても「あれやって」で通じる。
- **もう少し：** 画面で **`/`** を打つと呼び出せる。
- **今日の場面：** 最後に `/idea-to-mockup` を呼んで、自分のアイデアの試作を作ります。

## Agentic loop（エージェンティック・ループ）

- **ひとことで：** AIが「考える→やる→結果を見る→直す」を**自分で繰り返す**こと。
- **たとえ：** 人が一手ずつ指示しなくても、自分で段取りして進める職人。
- **今日の場面：** 「予約サイト作って」の一言から、Claudeが自分で手順を回していくのがこれ。だから**止める・確認する**操作が大事。

## ユニットテスト

- **ひとことで：** 部品ひとつが正しく動くかの自動チェック。
- **たとえ：** 料理を出す前に、ソースだけ味見してみること。
- **今日の場面：** STEP5。「この関数、ちゃんと動く？」を自動で確認。

## E2Eテスト（イーツーイー）

- **ひとことで：** お客さんの一連の流れを、丸ごと最初から最後まで確認するテスト。
- **たとえ：** 開店前に、店員が“お客さん役”で予約から完了まで通しでリハーサルする。
- **もう少し：** End to End（端から端まで）の略。
- **今日の場面：** STEP6。「フォーム入力→送信→完了画面」を自動で通し確認。

## Playwright（プレイライト）

- **ひとことで：** ブラウザを自動で操作するロボット。
- **たとえ：** お客さんの代わりに、画面をクリック・入力してくれる代行さん。
- **今日の場面：** E2Eテストで“お客さん役”を演じてもらいます。

## Playwright MCP

- **ひとことで：** Playwright（ブラウザ操作）を、MCP経由でClaudeにつないだもの。
- **たとえ：** AIに「ブラウザを動かす手」を生やす差込口。
- **今日の場面：** ハンズオン①で、Googleマップを自動で見て回り、お店情報を集めて表（CSV）にします。動く様子が画面で見えます。

## Claude Code のセッション

- **ひとことで：** ひとつの作業のまとまり（会話のひとかたまり）。
- **たとえ：** Claudeとの“ひとつの打ち合わせ”。その間は文脈（前の話）を覚えていてくれる。
- **もう少し：** 新しいセッションを始めると、前の会話はリセットされる（＝まっさらから）。
- **今日の場面：** 「予約サイトを作りたい」と伝えてから完成まで、ひとつのセッションで進めます。

## おまけ：今日のために入れる道具

### Docker（ドッカー）

- **ひとことで：** アプリを“箱（コンテナ）”ごと、そのまま動かす仕組み。
- **たとえ：** 中身（アプリ＋必要なもの）が密封された箱が届き、**スイッチを入れるだけで動く**。一からの面倒な「インストール」が要らない。
- **もう少し：** PC（Mac/Windows）の違いで「自分だけ動かない」が起きにくい。
- **今日の場面：** ハンズオン②の予約システムで、**データベース（PostgreSQL）や Go** を動かすのに使う。Node.js / npm は Claude が各自のPCに入れてくれるのでDocker不要。**アプリは開いて動かしたまま**に（閉じると中の箱も止まる）。

### ngrok（エングロック）

- **ひとことで：** 自分のPCに“公開の住所（URL）”を一本だけ通す道具。
- **たとえ：** 自宅で開いている店に、外から来られる**臨時の入口**を作る。
- **もう少し：** 無料版のURLは `〜.ngrok-free.app`。**一時的**で、PCやngrokを閉じると止まる。
- **今日の場面：** STEP8。これで作った予約サイトのURLを友達に送れます。事前に登録して **authtoken** を取得しておきます。
