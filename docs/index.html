1. 準備：今日使う道具をそろえる

① 事前に各自で済ませておくこと（詰まりやすい3点＋Win）

  • 有料のClaudeプラン（Pro / Max / Team / Enterprise のどれか）。※無料プランではCodeタブが使えません。
  • Claudeアカウントでログインできる状態。
  • ngrokのアカウント＋authtoken（取り方は下の④）。
  • Windowsの人は Git も。これは git を使うためではなく、Claude Code がコマンドを動かす土台として必要（Macは多くが標準装備）。Git for Windows を入れておく。

② Claude Desktop アプリを入れる

今日はターミナル（黒い画面）を使いません。Claude Desktop アプリの「Code」タブから操作します。

  • Mac：公式サイトから Claude Desktop（Apple Silicon / Intel 兼用）をダウンロードしてインストール。
  • Windows：x64 版（または ARM64 版）のインストーラをダウンロードしてインストール。※ Linux はDesktop非対応。
  • アプリを起動 → Anthropicアカウントでサインイン → 上部の 「Code」タブを開く。
  • 使うとき：Local を選ぶ → Select folder（作業フォルダ）を選ぶ → やりたいことを文章で書く → 出てきた差分（diff）どこが変わるかの表示を見て Accept / Reject。

Desktop には Claude Code が同梱なので、Node.js や CLI を別途入れる必要はありません（ただし Docker は別途必要）。

③ Docker Desktop を入れて、起動しておく

Dockerアプリを“箱（コンテナ）”ごと、そのまま動かす仕組み

  • ふつう、ソフトは使う前に面倒なインストールが必要。Dockerなら必要なものが箱に入った状態で来て、動かすだけ。
  • 今日は、ハンズオン②の予約システムで データベース（PostgreSQL）や Go を動かすのに使います。
  • Docker Desktop を公式サイトからインストールし、アプリを開いて起動したままにしておく（閉じると中の箱も止まります）。

※ Node.js / npm は Claude が各自のPCに入れてくれるので、そちらはDocker不要です。

④ ngrok の authtoken を取る

  1. ngrokにサインアップ（無料）。
  2. ダッシュボードの Your Authtoken ページで自分のトークンをコピー。
  3. あとで使うときは、次のコマンドで登録します（当日はClaudeに頼んでもOK）：
ngrok config add-authtoken ここに自分のトークンを貼る

公開するときは ngrok http 3000 のように「アプリが動いているポート番号」を指定します。無料版のURLは 〜.ngrok-free.app で、PCやngrokを閉じると止まる一時的なものです。

2. 最低限の地図（言葉と仕組み）

用語は「使う直前に1個ずつ」。全部覚える必要はありません。目標は「説明できる」ではなく「見て驚かない」。

そもそもClaude Codeって何？

LLMネットの文章を大量に読んだ“物知りな助手”

Claudeその助手のひとり（Anthropic社製）

Claude Codeその助手に“手”を付けたもの

ふつうのAIは「答えを言う」だけ。Claude Codeは、あなたのPCを実際に操作して、ファイルを作ったり動かしたりできる。

なぜ今、こんなに急に広まった？

  • 言葉で頼める：呪文のようなコードを覚えなくていい
  • 自分で手を動かす：提案だけでなく、実際に作って・試して・直す
  • 失敗を自分で直す：エラーが出たら原因を探して修正
  • 道具につながる：MCPで世界中のサービスと接続
要するに「作れる人の条件」が、“コードが書ける”から“やりたいことを言葉にできる”に変わった。

Webサービスを“レストラン”でイメージ

Webの仕組みは、ぜんぶレストランにたとえると分かりやすいです。

  • 客席・メニュー＝フロントエンド見える画面
  • ウェイター＝API注文を運ぶ窓口
  • 厨房＝バックエンド実際の調理＝処理
  • 食材庫＝データベースデータの保管
  • 店舗＝サーバー動かす場所そのもの / 店を借りる＝クラウドサーバー
今日の「サーバー」は、あなたのPCの中で一時的に動くだけ。クラウド契約も難しい設定もナシ。ngrokPCに“公開の住所”を一本だけ通す道具で、友達がスマホからアクセスできます。

Claude Codeの得意・苦手

得意：小さなWebアプリ・ツール・自動化をゼロから形にする／既存のものの修正・説明／調べ物や面倒な手作業の自動化／とりあえず動く試作（モック）を速く作る。

苦手：あいまいな丸投げ、正確さが命の計算を検算なしで使うこと、最新すぎる情報。→ 具体的に頼む・結果を自分で確認するのが大事。

3. Claude Codeの基本機能

画面の使い方（Claude Desktop）

  • Codeタブ → Local → 作業フォルダを選ぶ
  • 頼みごとを書くと、Claudeが変更案を出す → 差分を見て Accept / Reject
  • Preview：作ったアプリをアプリ内で起動して見られる
  • ポイント：いつでも止められる・言い直せる。気軽に試してOK。

remote-control（リモート操作）

remote-controlスマホからPCのClaude Codeに指示を送る

外出先からスマホで「これやっといて」と投げると、PC側のClaude Codeが受け取って作業します。“PCの前にいなくても進む”感覚。※新しめの機能。今日は紹介まで。

MCP（コネクタ）

MCPAI用の“USB端子”。外部サービスに挿す仕組み

Claude Code単体は「自分のPCの中」が基本。MCPを挿すと、外の世界（地図・メール・カレンダー等）につながります。

活用例（今日はやらない）：Gmailを読み取って下書き／Slackの返信案／Googleカレンダーで予定把握。メールやチャットの自動送信は事故のもとなので、実運用は「下書きまで＝人が最後に確認」が安全。

Skills（スキル）

Skillsよく使う“作業手順書”をまとめたパック

「この手順でやって」と覚えさせておけるもの。画面で / を打つと呼び出せます。今日は最後に /idea-to-mockup を使います。

Agentic loop（自分で回す仕組み）

Agentic loop考える→やる→結果を見る→直す、を自分で繰り返す

人が一手ずつ指示しなくても、ある程度自走します。だからこそ「止める・確認する」操作が大事になります。

4. ハンズオン①：AIにブラウザで調べ物 → CSV

Playwright MCP人の代わりにブラウザを自動操作する道具

  1. Claude Codeに Playwright MCP をつなぐ
  2. 「○○を調べて一覧にして」とお願いする
  3. Claudeがブラウザを自動で動かして集める（画面で見える！）
  4. CSV表計算で開ける、カンマ区切りの表ファイルに書き出す
  5. スプレッドシート（Excel / Googleスプレッドシート）で開く

頼み方の例（コピペ）：

Googleマップで“渋谷 カフェ”を調べて、上位10件の店名・住所・評価を集めて、cafes.csv に保存して
マナー：少量・学習目的で。相手サイトの規約・回数制限に配慮（大量取得はしない）。うまくいかない時は件数を減らす／条件を具体的にする。

5. ハンズオン②：予約サイトを作って公開（STEP0〜9）

ここが今日の主役。お店の予約サイトを作って、最後に ngrok で公開します。

これから作るもの

  • お店のトップページ（名前・写真・営業時間）
  • 予約フォーム（名前・日時・人数）
  • 送信された予約の一覧表示（管理者だけが見られる）
進め方：下のプロンプトを上から1つずつコピペして送る。1ステップごとに「動く状態」にして、差分を見て Accept。中身が分からなくてOK、コピペで大丈夫。

前提：Docker Desktop を起動しておく／Codeタブ→Localで作業フォルダを選ぶ／ngrokのauthtoken取得済み。

STEP 0はじめる（方針と技術スタックを決める）

お店の予約サイトを一緒に作りたいです。最後は ngrok で友達に公開します。 これから1ステップずつ指示します。勝手に先へ進めず、各ステップごとに「動く状態」にして、 変更した点と確認方法を短く教えてください。 技術スタックはこれで統一してください： - バックエンド: Go + Gin + GORM、データベースは PostgreSQL - フロントエンド: Next.js(React, TypeScript) + Tailwind CSS + shadcn/ui 一覧表は TanStack Table、入力チェックは Zod - APIは OpenAPI で定義し、フロントの型は openapi-typescript で自動生成して使う - ローカルの PostgreSQL（と、必要なら MinIO）は Docker(docker compose) で起動 - 画像などのファイル保管が必要になったら MinIO(S3互換) を使う まずは、この構成でプロジェクトの骨組み（フォルダ構成・docker compose・起動方法）だけ作って、 README に「起動手順」を書いてください。画面や機能はまだ要りません。
フォルダ構成ができ、docker compose up で PostgreSQL が立ち上がる。READMEに起動手順がある。

STEP 1トップページ（フロントエンド）

お店のトップページを作ってください。表示する内容は、 お店の名前 / かんたんな紹介文 / 写真（ダミーでOK）/ 営業時間 です。 Next.js + Tailwind + shadcn/ui を使い、スマホでも見やすいデザインにしてください。
ブラウザでトップページが表示され、スマホ幅でも崩れない。

STEP 2予約フォーム

トップページから行ける「予約フォーム」のページを作ってください。 入力項目：お名前 / 電話番号 / 希望日時 / 人数 / メモ(任意)。 入力チェックは Zod で行い、未入力や不正な値は分かりやすくエラー表示してください。 まだ送信先(バックエンド)は無くてよいので、送信したら画面に「予約を受け付けました」と 出るところまでで大丈夫です。
フォームに入力でき、空欄や変な値だとエラーが出る。送信で完了メッセージが出る。

STEP 3予約を受け取って保存（バックエンド / API / データベース）

予約フォームの内容を、バックエンド(Go + Gin)で受け取って PostgreSQL に保存できるように してください。 - GORM で reservations テーブルを用意（名前 / 電話 / 希望日時 / 人数 / メモ / 受付日時） - API は OpenAPI で定義し、フロントの型は openapi-typescript で生成して使う - DB は docker compose で起動している前提 予約フォームから送信したら、実際に DB に保存されるところまでつないでください。 保存できているかを確認する方法も教えてください。
フォーム送信した予約が、DBに1件保存されている。

STEP 4予約一覧で確認（“動いた！”の瞬間）

保存された予約を一覧で見られる管理ページ（/admin）を作ってください。 一覧は TanStack Table を使い、希望日時で並べ替えでき、新しい順で表示してください。 予約フォームから入れた予約が、この一覧にどんどん増えていくのを確認できる状態にしてください。
予約を入れる→ /admin の一覧に増える。並べ替えが効く。

STEP 5管理者ログイン（一覧は管理者だけに）

さきほどの予約一覧(/admin)は、管理者だけが見られるようにログイン機能を追加してください。 - 管理者は メールアドレス + パスワード でログイン - パスワードは必ずハッシュ化して保存（平文では保存しない） - ログインしていない人が /admin を開いたら、ログイン画面に飛ばす - ログアウトもできるようにする 管理者アカウントの「新規作成(サインアップ)」ページは、一旦は誰でも作成できる簡易版でOKです。 ※あとで「最初の1人だけ」「招待制」などに制限できるよう、その箇所にコメントを残しておいてください。
サインアップ→ログインで /admin が見られる。ログアウトすると /admin はログイン画面に戻される。

STEP 6ユニットテスト（部品ごとの自動チェック）

ユニットテスト部品ひとつが正しく動くかの確認（＝料理を出す前のソースの味見）。

ここまでの大事な部分にユニットテストを書いてください。最低でも次の3つ： 1) 予約の保存（バックエンド） 2) 入力チェック（Zod のバリデーション） 3) ログインの認証（正しいパスワード / 間違ったパスワード） テストの実行コマンドと、全部通ったことを確認する方法を教えてください。
テスト実行コマンドで、すべてのテストが PASS する。

STEP 7E2Eテスト（お客さん目線の通し確認 / Playwright）

E2Eテストお客さんの一連の流れを丸ごと確認（＝開店前の通しリハーサル）。Playwrightお客さんの代わりに画面を操作するロボットが演じます。

お客さんと管理者の一連の流れを、Playwright で自動テスト(E2E)してください。 1) お客さんが予約フォームから予約 → 完了メッセージが出る 2) 管理者がログイン → その予約が一覧に出ていることを確認 テストの実行方法も教えてください。
Playwright のテストが通り、予約〜一覧表示までが自動で確認できる。

STEP 8自分のPCで起動（Previewでも確認）

このアプリを自分のPCで起動して、ブラウザ（と Preview）で動いているのを確認したいです。 docker compose を含めた「起動手順」を最新の状態に直して、 最短で立ち上げる手順を 3〜5 行で教えてください。
手順どおりに起動でき、トップ→予約→管理ログイン→一覧まで一通り触れる。

STEP 9ngrokで公開（友達に送れるURLにする）🎉

いま動いているアプリを ngrok で一時的に公開して、友達がスマホからアクセスできる URL を作ってください。出てきた 〜.ngrok-free.app の URL を教えてください。 （PCやngrokを閉じると止まる、という点も確認したいです）
〜.ngrok-free.app のURLが出て、スマホのブラウザから予約フォームが開ける。
無料版のURLは一時的。PCやngrokを閉じると止まります（＝“今この瞬間に生きているリンク”）。常時公開したいなら、本当はサーバーに置きます（今日はやらない）。

6. うまくいかない時のコツ

  • 止める：暴走しそうなら停止ボタン
  • 言い直す：「さっきのを、こう変えて」でOK
  • 差分を見る：何が変わるか必ず確認してからAccept
  • 小さく頼む：一度に大きく頼まず、1ステップずつ
  • エラーは会話のきっかけ：出たエラーをそのまま貼って「このエラーを直して」
  • 進めすぎたら「STEP◯の状態に戻して」
