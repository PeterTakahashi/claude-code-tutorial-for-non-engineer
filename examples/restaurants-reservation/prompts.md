# 予約サイト：STEP別コピペ用プロンプト

上から順に **1つずつ** Claude Code に送ってください。各ステップは「ここまでで動く」状態を目指します。
各STEPの「✅ 確認」まで動いたら、次のSTEPへ。

---

## STEP0：はじめる（方針と技術スタックを決める）

```
お店の予約サイトを一緒に作りたいです。最後は ngrok で友達に公開します。
これから1ステップずつ指示します。勝手に先へ進めず、各ステップごとに「動く状態」にして、
変更した点と確認方法を短く教えてください。

技術スタックはこれで統一してください：
- バックエンド: Go + Gin + GORM、データベースは PostgreSQL
- フロントエンド: Next.js(React, TypeScript) + Tailwind CSS + shadcn/ui
  一覧表は TanStack Table、入力チェックは Zod
- APIは OpenAPI で定義し、フロントの型は openapi-typescript で自動生成して使う
- ローカルの PostgreSQL（と、必要なら MinIO）は Docker(docker compose) で起動
- 画像などのファイル保管が必要になったら MinIO(S3互換) を使う

まずは、この構成でプロジェクトの骨組み（フォルダ構成・docker compose・起動方法）だけ作って、
README に「起動手順」を書いてください。画面や機能はまだ要りません。
```

✅ 確認：フォルダ構成ができ、`docker compose up` で PostgreSQL が立ち上がる。READMEに起動手順がある。

---

## STEP1：トップページ（フロントエンド）

```
お店のトップページを作ってください。表示する内容は、
お店の名前 / かんたんな紹介文 / 写真（ダミーでOK）/ 営業時間 です。
Next.js + Tailwind + shadcn/ui を使い、スマホでも見やすいデザインにしてください。
```

✅ 確認：ブラウザでトップページが表示され、スマホ幅でも崩れない。

---

## STEP2：予約フォーム

```
トップページから行ける「予約フォーム」のページを作ってください。
入力項目：お名前 / 電話番号 / 希望日時 / 人数 / メモ(任意)。
入力チェックは Zod で行い、未入力や不正な値は分かりやすくエラー表示してください。
まだ送信先(バックエンド)は無くてよいので、送信したら画面に「予約を受け付けました」と
出るところまでで大丈夫です。
```

✅ 確認：フォームに入力でき、空欄や変な値だとエラーが出る。送信で完了メッセージが出る。

---

## STEP3：予約を受け取って保存（バックエンド / API / データベース）

```
予約フォームの内容を、バックエンド(Go + Gin)で受け取って PostgreSQL に保存できるように
してください。
- GORM で reservations テーブルを用意（名前 / 電話 / 希望日時 / 人数 / メモ / 受付日時）
- API は OpenAPI で定義し、フロントの型は openapi-typescript で生成して使う
- DB は docker compose で起動している前提
予約フォームから送信したら、実際に DB に保存されるところまでつないでください。
保存できているかを確認する方法も教えてください。
```

✅ 確認：フォーム送信した予約が、DBに1件保存されている（確認方法どおりに見られる）。

---

## STEP4：予約一覧で確認（“動いた！”の瞬間）

```
保存された予約を一覧で見られる管理ページ（/admin）を作ってください。
一覧は TanStack Table を使い、希望日時で並べ替えでき、新しい順で表示してください。
予約フォームから入れた予約が、この一覧にどんどん増えていくのを確認できる状態にしてください。
```

✅ 確認：予約を入れる→ /admin の一覧に増える。並べ替えが効く。

---

## STEP5：管理者ログイン（一覧は管理者だけが見られるように）

```
さきほどの予約一覧(/admin)は、管理者だけが見られるようにログイン機能を追加してください。
- 管理者は メールアドレス + パスワード でログイン
- パスワードは必ずハッシュ化して保存（平文では保存しない）
- ログインしていない人が /admin を開いたら、ログイン画面に飛ばす
- ログアウトもできるようにする

管理者アカウントの「新規作成(サインアップ)」ページは、一旦は誰でも作成できる簡易版でOKです。
※あとで「最初の1人だけ」「招待制」などに制限できるよう、その箇所にコメントを残しておいてください。
```

✅ 確認：サインアップ→ログインで /admin が見られる。ログアウトすると /admin はログイン画面に戻される。

---

## STEP6：ユニットテスト（部品ごとの自動チェック）

```
ここまでの大事な部分にユニットテストを書いてください。最低でも次の3つ：
1) 予約の保存（バックエンド）
2) 入力チェック（Zod のバリデーション）
3) ログインの認証（正しいパスワード / 間違ったパスワード）
テストの実行コマンドと、全部通ったことを確認する方法を教えてください。
```

✅ 確認：テスト実行コマンドで、すべてのテストが PASS する。

---

## STEP7：E2Eテスト（お客さん目線の通し確認 / Playwright）

```
お客さんと管理者の一連の流れを、Playwright で自動テスト(E2E)してください。
1) お客さんが予約フォームから予約 → 完了メッセージが出る
2) 管理者がログイン → その予約が一覧に出ていることを確認
テストの実行方法も教えてください。
```

✅ 確認：Playwright のテストが通り、予約〜一覧表示までが自動で確認できる。

---

## STEP8：自分のPCで起動（Previewでも確認）

```
このアプリを自分のPCで起動して、ブラウザ（と Preview）で動いているのを確認したいです。
docker compose を含めた「起動手順」を最新の状態に直して、
最短で立ち上げる手順を 3〜5 行で教えてください。
```

✅ 確認：手順どおりに起動でき、トップ→予約→管理ログイン→一覧まで一通り触れる。

---

## STEP9：ngrokで公開（友達に送れるURLにする）🎉

```
いま動いているアプリを ngrok で一時的に公開して、友達がスマホからアクセスできる
URL を作ってください。出てきた `〜.ngrok-free.app` の URL を教えてください。
（PCやngrokを閉じると止まる、という点も確認したいです）
```

✅ 確認：`〜.ngrok-free.app` のURLが出て、スマホのブラウザから予約フォームが開ける。

## うまくいかない時のひとこと（共通）

- エラーが出たら、その内容をそのまま貼って **「このエラーを直して」**。
- 大きく変わりそうなら **「一旦止めて、何をするか先に説明して」**。
- 進めすぎたら **「STEP◯の状態に戻して」**。
