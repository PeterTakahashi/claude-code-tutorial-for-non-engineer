## 0. これは何か

- 自宅のイベントスペースで開く **4時間の対面 Claude Code 勉強会** の教材一式。
- 参加者は **9名、全員ノンエンジニア**。主催・進行は Seiya（自学のエンジニア）。
- 目的は「エンジニアにする」ことではなく、**AIに頼んで自分のアイデアを“動く形”にする**体験をしてもらうこと。

### 当日のゴール（参加者の成果物）
**自分のPCで動く「お店の予約サイト」を作り、ngrok で“友達に送れるURL”にするところまで。**
クラウドのサーバー契約・デプロイ・独自ドメインはやらない。「サーバー」は各自PC内で一時的に動くだけ＋ngrokで公開の住所を一本通す。

## 1. 教え方の方針（ここをブラさない）

- **用語は「前もって全部」でも「一切なし」でもなく、“使う直前に1個ずつ”。** 1語ごとに一言の言い換え＋たとえを添える。
- スライドではこれを **「用語ピル＋言い換え」** の形で統一表現している（後述の `.term` / `.gloss`）。新しい用語を出すときも必ずこの形にすること。
- Web構成は **レストランのたとえ** で説明：客席/メニュー＝フロントエンド、ウェイター＝API、厨房＝バックエンド、食材庫＝データベース、店舗＝サーバー、店を借りる＝クラウド。
- トーン：「**全部わからなくてOK。動けば勝ち**」。エラーは失敗ではなく“会話のきっかけ”。
- ゴールは「用語を説明できる」ではなく「**見て驚かない**」。

## 2. 環境（決定事項）

- Claude Code は **Claude Desktop アプリの「Code」タブ**から使う（ターミナルを使わない＝非エンジニアに優しい）。
- **macOS / Windows（x64・ARM64）対応、Linux非対応。**
- **Code タブは有料プラン（Pro / Max / Team / Enterprise）が必要**（参加者各自）。
- **Windows はローカルセッションに Git のインストールが必要**（Macは多くが標準装備）。
- **Docker（Docker Desktop）のインストールが必要**。用途はハンズオン②の予約システムで **PostgreSQL（DB）や Go** を動かすため（Playwright/MCPのためではない）。**Node.js / npm は Claude が各自PCにローカルインストール**するのでDocker不要。当日は開いて起動したままにしておく（閉じると中のコンテナも止まる）。※料理・厨房の比喩は「厨房＝バックエンド」と被るので使わない。
- Desktop には Claude Code が同梱＝**Node.js や CLI の別途インストール不要**（ただし Docker は別途必要）。差分のAccept/Reject、Preview、`/` でのSkills呼び出しが使える。
- セットアップは**当日**実施。ただし詰まりやすい **「有料Claudeプラン・Claudeアカウント・ngrokアカウント＆authtoken」の3点は事前に各自で済ませてもらう**前提。
- ※ Desktop の正確な手順・ngrok の authtoken 取得手順は、**手順書を書くときに最新ドキュメントで再確認**すること（記憶でハードコードしない）。

## 4. 教える内容（明示された範囲）

**概念パート**
- なぜ Claude Code が急に普及したか／Skillsとは／MCPとは／Claude Codeの得意・苦手分野
- 基本機能：画面の使い方、**remote-control**（スマホからPCのClaude Codeに指示。新しめの機能なので**概念紹介に留める**）、**MCP（コネクタ）**、**Skills**、**Agentic loop**

**ハンズオン①（やる）：Playwright MCP で調べ物 → CSV → スプレッドシート**
- Playwright MCP（人の代わりにブラウザを自動操作）を有効化
- 例「Googleマップで“渋谷 カフェ”上位10件の 店名・住所・評価 を集めて `cafes.csv` に保存して」→ スプレッドシートで開く
- マナー注意：**少量・学習目的**で。相手サイトの規約・回数制限に配慮（大量取得しない）。

**MCP活用例（説明のみ・ハンズオンしない）**
- Gmail を読み取り→**自動返信（実運用は“下書きまで”が安全）**
- Slack を読み取り→返信案
- Googleカレンダーを読み込み→予定把握

**ハンズオン②（やる）：予約サイト → ngrok 公開（STEP0〜8）**
0. 「お店の予約サイトを作りたい」と伝える（セッション/Agentic loop）
1. トップページ（フロントエンド）
2. 予約フォーム
3. 予約を受け取って保存（バックエンド/API/データベース）
4. 予約一覧で確認 ← “動いた！”
5. ユニットテスト
6. E2Eテスト（Playwright）
7. 自分のPCで起動（Previewでも確認）
8. **ngrokで公開 → URLを友達に送る** 🎉
- ngrok注意：無料版URLは `〜.ngrok-free.app` で**一時的、PC/トンネルを閉じると止まる**。

**カバーする用語リスト**：LLM / コンピュータ / インターネット / API / データベース / バックエンド / フロントエンド / クラウドサーバー / サーバー / Claude / Git / GitHub / CLI / GPU / コマンド / MCP / Skills / Agentic loop / ユニットテスト / E2Eテスト / Playwright / Playwright MCP / Claude Codeのセッション
（Git/GitHub/デプロイは**必須ハンズオンから外し**、地図で名前だけ触れる扱い。）

### slides.md（Marp）
- `theme: default` ＋ 先頭の `<style>` でカスタムテーマを定義。
- 構成：イントロ（自己紹介テンプレ/ゴール/流れ/完成形デモ予告）→ Part1 最低限の地図 → Part2 基本機能 → ハンズオン① → ハンズオン② → まとめ。
- 自己紹介スライドは**書き換え前提のテンプレ**。

### idea-to-mockup.skill
- 非エンジニアの「○○が欲しい」を、**やさしい対話で質問→動くモック**まで作るSkill。
- 質問は1〜3問ずつ・言い換え付き・迷ったらデフォルトで前進。
- **親分岐は「誰が使う？」**（自分だけ/特定の仲間/不特定多数）。自分だけ用途ならログイン・サーバー・ドメイン・課金等を丸ごとスキップ。
- モックの既定は**外部アカウント・支払い不要で即動くWebアプリ＋ダミーデータ**。有料API・認証・公開は「必要」と言われた時だけ段階的に。
- 当日は締めで各自が `/idea-to-mockup` を呼んで自分のアイデアを試す想定。

## 6. スライドのデザインシステム（編集時は厳守）

`slide/slides.md` 先頭の `<style>` に定義済み。**新しいスライドや要素を足すときも、この語彙だけで組むこと**（崩さない）。

- 色：`--ink #1F2933` / `--paper #FAF9F6` / `--accent`（クレイ＝Claude色）`#D97757` / `--accent2`（ティール＝行動）`#2A9D8F` / `--muted #6B7280` / `--line #E7E3DC` / `--soft #F0EDE7`
- スライド種別クラス：`section.title`（表紙）/ `section.section`（章扉・濃紺背景）
- 部品クラス：
  - `.eyebrow`（小見出しラベル）
  - `.term`（用語ピル）＋ `.gloss`（言い換え・グレー）← **用語紹介の基本形**
  - `.do`（ティールの囲み＝やること/ポイント）/ `.warn`（クレイの囲み＝注意）
  - `.cols`＋`.col`（横並びカード）/ `.diagram`＋`.box`＋`.arrow`（流れ図）/ `.small`
- 用語を出す基本形：
  `<span class="term">用語</span><span class="gloss">一言の言い換え</span>`
- 章扉と表紙はページ番号を消す：`<!-- _paginate: false -->`

### Marp 書き出し / プレビュー
```
marp slide/slides.md -o slide/slides.pdf       # PDF（要Chrome/Chromium）
marp slide/slides.md -o slide/slides.pptx      # PowerPoint
marp slide/slides.md --preview                 # プレビュー
marp -s slide                                  # 保存ごと自動リロード（編集向き）
```
※ 非対話環境でstdin待ちになる場合は `--no-stdin` を付ける。
